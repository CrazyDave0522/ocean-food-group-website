# Design: Style Legal Pages

## Architecture

The legal pages (`/privacy`, `/terms`) render markdown content using the `marked` library. To avoid affecting other markdown content on the site, we'll create a new scoped CSS class `.prose-legal` that targets common HTML elements generated by the markdown parser.

```
User requests /privacy or /terms
         ↓
     Page component (app/privacy/page.tsx, app/terms/page.tsx)
         ↓
     Reads .md file from public/legal/
         ↓
     Parses markdown → HTML via marked
         ↓
     Sanitizes with DOMPurify
         ↓
     Renders in <div className="prose-legal">
         ↓
     Styles applied via styles/components/legal-content.css
```

## Styling Strategy

### 1. Typography Scale

Map markdown heading levels to existing font-size tokens:

| Markdown | Element | Token           | Desktop | Mobile | Notes                              |
|----------|---------|-----------------|---------|--------|-----------------------------------|
| #        | h1      | --fs-h1         | 32–48px | 28–36px | Page title (rarely used in legal) |
| ##       | h2      | --fs-h2         | 24–36px | 20–28px | Major sections                    |
| ###      | h3      | --fs-h3         | 20–28px | 18–24px | Subsections                       |
| ####     | h4      | --fs-h4         | 18–24px | 16–20px | Sub-subsections                  |
| text     | p       | --fs-body       | 14–16px | 13–16px | Body copy                        |

**Implementation:**
- Use `font-size: var(--fs-h2)` for `h2`, etc.
- Add `margin-top: var(--space-xl)` and `margin-bottom: var(--space-md)` to headings for breathing room
- Set `line-height: 1.6` for body paragraphs; `1.4` for headings
- Use default `letter-spacing` throughout (no special adjustments)

### 2. Visual Hierarchy

- **h2** (Section headers): Bold, larger size, top margin creates breaks between sections
- **h3** (Subsections): Medium size, slightly less prominent than h2
- **h4** (Minor headers): Smaller, same color as body text but bolded
- **Paragraphs**: Line height ~1.6–1.8 for readability; max-width-aware for long lines

### 3. List Styling

- **Unordered lists (`<ul>`)**:
  - Bullet points (default `disc`)
  - `margin-left: 1.5em` or use Tailwind's `pl-*`
  - `line-height: 1.6` for list items
  - `margin-bottom: 0.5em` between items

- **Ordered lists (`<ol>`)**:
  - Numbered (default `decimal`)
  - Same indentation and spacing as `<ul>`

**Special case:** Legal content often has nested lists (e.g., sections with sub-points); ensure proper visual nesting via indentation.

### 4. Inline Elements

- **Links (`<a>`)**:
  - Color: `var(--color-primary)` (#111827)
  - Underline: `underline`, `underline-offset-2`
  - Hover: `opacity-80` or `opacity-60` transition

- **Strong (`<strong>`)**:
  - Font-weight: 700 (already bold)
  - No color change

- **Emphasis (`<em>`)**:
  - Font-style: italic
  - No color change

- **Code (`<code>`, if present)**:
  - Monospace font (fallback to system mono)
  - Light gray background: `bg-slate-100`
  - Padding: `0.2em 0.4em`
  - Slight border radius: `rounded-sm`

### 5. Container & Spacing

- **Max-width**: Leverage existing container logic or add `max-w-prose` (≈65ch) to the wrapper
- **Horizontal padding**: Use Tailwind's responsive padding (`px-4` on mobile, `px-6`/`px-8` on desktop) or CSS `padding-inline: var(--pad-inline)`
- **Vertical padding**: Keep existing `py-12` from page component
- **Color**: Black text (`--color-text: #111111`) on white background

### 6. Mobile Optimization

- Font sizes already use `clamp()` in tokens (responsive by design)
- Headings: Ensure line-breaking works well; test on ~375px screens
- Lists: Reduce left indentation if needed on mobile to preserve space
- Links: Ensure sufficient touch target (≥44px height)

## Future Enhancements

### Disclaimer Boxes (Post-MVP)

Legal documents often contain high-importance disclaimers (e.g., allergen warnings, liability limits). This could be addressed in a future iteration with a special `.prose-disclaimer` class for styled callout boxes. Deferred to maintain focused initial scope.

## Color Palette

- **Text**: `--color-text: #111111` (primary text)
- **Links**: `--color-primary: #111827` (brand navy)
- **Hover**: Same with opacity change or lighter shade
- **Background**: White (default)
- **Accents** (if disclaimer boxes added): `amber-50` (light) / `amber-500` (border)
